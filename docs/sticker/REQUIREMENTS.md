# REQUIREMENTS: 도담 발자국 — v3.5 (12종)

## 기능 요구사항

### US-1: 발자국 컬렉션 페이지
- **역할**: 로그인 사용자
- **동작**: 12종 발자국을 한 화면에 4열×3행 그리드로 표시. 달성(👣)/미달성(🔒). "도담 N걸음" 진행률.
- **위치**: `/footprints` (설정에서 "내 발자국" 메뉴로 진입)
- **페이지 진입 시**: `get_quest_progress` 호출 → 미달성 발자국 중 조건 충족한 것 자동 달성 처리

### US-2: 발자국 자동 달성 (패시브)
- **역할**: 시스템
- **트리거**: 컬렉션 페이지 진입 시 자동 체크
- **동작**: progress와 FOOTPRINTS 상수 비교 → 달성 조건 충족 시 `achieve_footprint` RPC → 토스트
- **대상**: 12종 전체 (쉬운 7종 + 도전 5종)
- **제약**: 동일 발자국 중복 달성 불가 (UNIQUE + ON CONFLICT DO NOTHING)

### US-3: 거래완료 시 즉시 체크 (액티브)
- **역할**: 시스템
- **트리거**: 채팅에서 거래완료 버튼 탭 후
- **동작**: 장터 관련 발자국(첫 출품, 첫 거래, 첫 나눔, 장터의 왕) 즉시 체크 + 달성 시 토스트

### US-4: 설정에서 진입
- **역할**: 로그인 사용자
- **동작**: 설정 페이지에 "내 발자국 👣" 메뉴 → 컬렉션 페이지 이동

## 비기능 요구사항

### NFR-1: 보안
- `achieve_footprint`는 SECURITY DEFINER → 클라이언트에서 직접 INSERT 불가
- `get_quest_progress`도 SECURITY DEFINER → 타인 진행 조회 불가

### NFR-2: 성능
- `get_quest_progress`는 9개 서브쿼리 → 컬렉션 진입 시 1회만 호출
- `user_profiles.footprint_ids` 캐시로 "도담 N걸음" 빠르게 계산
- 거래완료 시 체크는 `get_quest_progress` 1회 + `achieve_footprint` 0~4회

### NFR-3: UX
- 컬렉션 진입 시 스켈레톤 로딩 (1초 이내 목표)
- 미달성: 🔒 + 흐린 처리 + 진행률 표시 (예: 3/10)
- 달성: 이모지 컬러풀 + 👣 마크
- 도전 발자국 미달성: 진행률(N/목표) 표시로 동기부여
- 토스트: 기존 dodam-toast 재사용
