# 육아 혜택 스캐너 (benefit.dodam.life) — 아이디어 브리프

> 아이디어 브리프 | 2026-04-08
> "아이 나이랑 거주지만 넣으면 받을 수 있는 육아 혜택 전부 + 연간 총 금액"

---

## Executive Summary

"아이 수, 나이, 거주지" 3개만 입력하면 받을 수 있는 **모든 육아 보조금/혜택을 한 화면에** 보여주고, **연간 총 수령 금액**을 계산해주는 마이크로 웹 서비스. 보조금24 API 1개로 하루 만에 만들 수 있고, 경쟁 서비스가 없으며, "돈"이라는 즉각적 가치로 바이럴 가능성 최고.

---

## 1. 왜 이게 제일 엣지 있는가

### 경쟁자 분석

| 기존 서비스 | 한계 | 우리 차별점 |
|------------|------|-----------|
| 보조금24 (정부24) | 범용 — 육아/노인/장애 다 섞임, 찾기 어려움 | **육아 특화**, 3초만에 전체 혜택 |
| 복지로 | UI가 구식, 검색 불편 | 모바일 최적화 카드 UI |
| 서울 몽땅정보 | 서울 한정 | **전국** |
| 맘카페 후기 | 파편화, 신뢰도 낮음 | **공식 데이터 기반 100% 정확** |

→ **"아이 나이 + 거주지 = 전체 혜택 + 연간 총액"을 해주는 서비스 = 없음**

### 엣지 요소

- **바이럴 킬러**: "나 이 앱 써봤는데 연 840만원 놓치고 있었어" → 맘카페 폭발 예상
- **즉각적 가치**: 돈 = 최고의 가치. 클릭 한 번에 신청 페이지로 이동
- **도담 시너지**: 아이 정보가 도담에 이미 있어서 자연스러운 연결

---

## 2. 서비스 스펙

### 핵심 기능

**입력 (3개 필드)**
- 아이 수 (1명/2명/3명+)
- 아이별 생년월일 (나이 자동 계산)
- 거주 지역 (시/군/구)

**출력**
- 해당하는 혜택 카드 리스트
- **연간 총 수령 금액 (큰 숫자로 상단에)** ← 바이럴 핵심
- 각 카드: 혜택명 + 월 금액 + 지원기간 + "신청하기" 버튼

### 부가 기능 (V2)

- "이미 받고 있어요" 체크 → **안 받고 있는 것만 빨간 하이라이트**
- "둘째 낳으면?" 시뮬레이션 → 추가 혜택 + 금액 표시
- 카카오톡 공유: "우리 가족 육아 혜택 연 ₩8,400,000" 카드 이미지

| API | 용도 | 비용 |
|-----|------|------|
| 보조금24 공공서비스 혜택 정보 | 중앙정부 보조금 전체 | **무료** |

- 인증: API키 발급 (무료)
- 응답: 서비스명, 혜택유형, 선정기준, 지원내용, 온라인신청URL

- 반응형 웹 (모바일 최적화)

### 인프라

- benefit.dodam.life 서브도메인
- Vercel 또는 도담 기존 서버

## 5. 사용자 시나리오

### 시나리오 1: 첫째맘 김서연 (30세, 서울 강남구, 아이 8개월)

1. benefit.dodam.life 접속
2. 아이 1명, 생년월일 2025-08-15, 서울 강남구 입력
3. 결과: **"연간 약 ₩15,800,000 받을 수 있어요!"**
   - 부모급여 ₩1,000,000/월 ✅ (이미 받는 중)
   - 아동수당 ₩100,000/월 ✅
   - 첫만남이용권 ₩2,000,000 ✅
   - **영아수당 ₩300,000/월 ❌ (안 받고 있음!)**
   - **서울시 출산축하금 ₩1,000,000 ❌ (안 받고 있음!)**
4. "영아수당 신청하기" 클릭 → 정부24로 이동

### 시나리오 2: 셋째맘 박지현 (35세, 경기 수원, 아이 3명)

1. 아이 3명 입력
2. 결과: **"연간 약 ₩22,400,000 + 다자녀 혜택 7개"**
   - 기본 혜택 + **다자녀 전기/가스/난방/KTX 할인** 추가 표시
3. "와 이거 다 받을 수 있었어?" → 맘카페에 공유
