# 도담 라이프 (dodam.life) — 옴니버스 마이크로서비스 포털

> 아이디어 브리프 | 2026-04-08

---

## Executive Summary

육아에 필요한 외부 데이터를 API 조합으로 끌어와 **매일 하나씩** 독립 웹 서비스로 개발하고, 최종적으로 **도담 라이프(dodam.life) 포털**에서 옴니버스 형태로 제공하는 마이크로서비스 시리즈.

---

## 1. 전략: 2단계 접근

```
Phase 1: 매일 하나씩 독립 웹 서비스 개발
         benefit.dodam.life → today.dodam.life → ...
         각각 독립적으로 동작, 독립적으로 바이럴

Phase 2: 포털로 통합
         dodam.life = 옴니버스 포털
         모든 마이크로서비스를 한 곳에서 접근
         "육아의 모든 것을 찔러서 가져오는 포털"
```

### 왜 이 순서인가

1. **리스크 최소화**: 하나씩 만들면서 반응 확인. 안 되면 버려도 됨
2. **매일 성취감**: 하루 1서비스 = 매일 런칭하는 쾌감
3. **포털은 나중에**: 콘텐츠(서비스)가 쌓인 후에 포털이 의미 있음
4. **옴니버스 구조**: 각 서비스가 독립적이되, 포털에서 하나의 세계관

## 4. 옴니버스 컨셉

영화 옴니버스처럼:
- 각 서비스 = **독립적인 단편** (혼자서도 완결)
- 포털 = **단편들을 엮는 세계관** (도담 라이프)
- 공통 요소 = **"아이 정보"** (나이, 거주지가 한번 입력되면 모든 서비스에 적용)

### 세계관 연결 포인트

| 공유 데이터 | 활용 서비스들 |
|------------|-------------|
| 아이 생년월일 | 혜택 스캐너, 예방접종, 성장곡선, 이유식, 급식 |
| 거주지 (시/군/구) | 혜택 스캐너, 소아과119, 안전지도, 감염병, 어린이집 |
| 아이 알레르기 | 급식 알리미, 이유식 영양사 |
| 아이 학교 | 급식, 학사일정, 학교정보 |

→ **한번 입력하면 15개 서비스 모두에 적용** = 포털의 진짜 가치

## 5. 기술 방향

### 개별 서비스 (Day 1~15)

- **각각 독립 Next.js 앱** 또는 **도담 기존 프로젝트 내 라우트**
- 서브도메인: xxx.dodam.life
- 공공 API 키 공유 (한 계정으로 여러 서비스)

### 포털 통합 (Day 16~20)

**Option A: 서브도메인 유지 + 포털은 링크 허브**
- dodam.life = 카드형 링크 모음
- 각 서비스는 서브도메인 그대로
- 장점: 독립성 유지, 장애 격리

**Option B: 모노리포 내 라우트**
- dodam.life/benefit, dodam.life/sos, ...
- 하나의 Next.js 앱에 모든 서비스
- 장점: 데이터 공유 쉬움, 공통 인증

→ **추천: Phase 1은 Option A (빠른 개발), Phase 2에서 Option B로 통합**
