# 매일 아침 육아 대시보드 — 킬러 번들 아이디어 브리프

> 아이디어 브리프 | 2026-04-08
> "아침에 한 번 열면 오늘 육아에 필요한 모든 것"

---

## Executive Summary

15개 마이크로서비스 중 **시너지가 가장 강력한 7개를 하나의 대시보드**로 통합. "매일 아침 여는 육아 앱" = 모든 도담 서비스의 **관문(gateway)**이자 **데이터 해자**.

개별 서비스는 "유용한 도구". 번들은 **"떠날 수 없는 생태계"**.

---

## 1. 왜 묶는가 — 엣지 분석

### 개별 vs 번들 비교

| 항목 | 개별 서비스 15개 | 킬러 번들 1개 |
|------|:---------------:|:------------:|
| 경쟁 장벽 | 각각 모방 가능 | **조합이 장벽** — 7개 API 파이프라인 카피 어려움 |
| 사용 빈도 | 서비스별 월 1~2회 | **매일 아침** (DAU 극대화) |
| 락인 | 약함 (대체재 많음) | **극강** — 알레르기+학교+건강 데이터 축적 |
| 데이터 해자 | 각 서비스에 분산 | **통합 프로필** — 이 조합의 데이터는 우리만 가짐 |
| 바이럴 | 각각 공유 | **"매일 아침 이거 보고 시작해"** — 습관 추천 |
| 네트워크 효과 | 없음 | **동네 기반** — 같은 학교/어린이집 부모끼리 |

### 핵심 인사이트

> **"아이 정보를 한번 입력하면 7가지 가치가 매일 아침 자동으로 나온다"**
>
> 이것이 단순 도구와 플랫폼의 차이.
> 개별 앱 7개를 따로 깔아서 아이 정보를 7번 입력할 사람은 없다.

## 2. 번들 구성 — 7개 서비스 통합

### 코어 대시보드 (매일 표시)

| # | 서비스 | 대시보드 역할 | 데이터 소스 |
|---|--------|-------------|-----------|
| 1 | **오늘 나갈까?** | 외출 점수 + 옷차림 | 에어코리아 + 기상청(단기예보+생활지수) |
| 2 | **급식 알레르기** | 오늘 급식 + 알레르기 경고 | NEIS 급식 + 식약처 영양성분 |
| 3 | **감염병 알림** | 우리 동네 감염병 현황 | 질병관리청 감염병 발생현황 |
| 4 | **혜택 스캐너** | 놓친 혜택 알림 | 보조금24 |

### 확장 섹션 (주간/이벤트 기반)

| # | 서비스 | 표시 조건 | 데이터 소스 |
|---|--------|----------|-----------|
| 5 | **예방접종 매니저** | 다음 접종일 D-14부터 표시 | 질병관리청 접종일정 + 심평원 소아과 |
| 6 | **안전지도** | 외출 점수 70+ 일 때 | 행안부 놀이시설 + 어린이보호구역 |
| 7 | **주말 어디갈까?** | 금요일~일요일에 자동 표시 | 관광공사 TourAPI + 축제 + 주차장 |

### 데이터 파이프라인

```
[한번 입력]
아이 생년월일 + 거주지(시/군/구) + 알레르기 + 학교/어린이집
         │
         ├──→ 외출 점수 (거주지 → 날씨+대기질)
         ├──→ 급식 알레르기 (학교 → NEIS 급식, 알레르기 → 매칭)
         ├──→ 감염병 (거주지 → 지역 감염병)
         ├──→ 혜택 스캐너 (아이 나이+거주지 → 보조금 스캔)
         ├──→ 예방접종 (생년월일 → 접종 일정 자동 생성)
         ├──→ 안전지도 (거주지 → 주변 놀이터/스쿨존)
         └──→ 주말 추천 (거주지+아이 나이 → 맞춤 나들이)
```

→ **입력 1번, 출력 7가지.** 이게 번들의 힘.

## 4. 엣지 4가지 상세

### 4.1 락인 효과 (Lock-in)

| 축적 데이터 | 시간 | 이탈 비용 |
|------------|------|----------|
| 아이 알레르기 프로필 | 첫 주 | 낮음 |
| 학교/어린이집 연동 | 2주차 | 중간 |
| 예방접종 기록 | 1개월 | 높음 |
| 12개월 외출/급식 히스토리 | 1년 | **매우 높음** |
| 둘째/셋째 아이 추가 | 지속 | **이탈 불가** |

**메커니즘**: 시간이 지날수록 **개인화 데이터가 축적** → 새 앱으로 옮기면 처음부터 다시 입력해야 함 → 안 옮김

### 4.2 원스톱 가치

**Before (현재):**
```
아침 기상 → 미세먼지 앱 확인 → 날씨 앱 확인 → 학교 앱에서 급식 확인
→ 네이버에서 감염병 검색 → 정부24에서 보조금 검색
= 15분 + 7개 앱
```

**After (번들):**
```
아침 기상 → 도담 아침 열기
= 30초 + 1개 앱
```

→ **시간 절약 = 매일 14분 30초 × 365일 = 연간 88시간 절약**

### 4.3 네트워크 효과

- **같은 학교 부모**: "우리 학교 급식 오늘 알레르기 성분 있대" — 부모 단톡방 공유
- **같은 동네 부모**: "우리 동네 수족구 급증이래" — 동네 맘카페 공유
- **같은 연령대**: "8개월이면 영아수당 신청해야 해" — 출산 동기 단톡방 공유

→ **사용자가 늘수록 콘텐츠(리뷰, 공유) 풍부** → 더 많은 사용자 유입

### 4.4 데이터 해자 (Moat)

```
[우리만 가지는 데이터 조합]

아이 프로필 ← 기본 정보
    ├── 건강 (알레르기, 접종, 질환)
    ├── 교육 (학교/어린이집, 급식 선호)
    ├── 생활 (거주지, 외출 패턴)
    └── 재정 (보조금 수급 현황)

→ 이 4차원 데이터를 동시에 가진 서비스 = 한국에 없음
→ 카카오/네이버가 카피하려면? 7개 공공 API 파이프라인 구축 + 사용자 데이터 재축적 필요
→ 데이터 축적에 최소 1년 → 선점 우위
```

## 7. 경쟁자 분석

| 기존 서비스 | 커버리지 | 우리 차별점 |
|------------|---------|-----------|
| 미세먼지 앱 (에어비주얼 등) | 대기질만 | 육아 맞춤 외출 판단 + 옷차림 |
| 학교 급식 앱 | 급식만 | 알레르기 매칭 + 영양 분석 |
| 질병관리청 앱 | 감염병만 | 우리 동네 + 아이 연령 맞춤 |
| 보조금24 | 전체 보조금 | 육아 특화 + 간편 3개 입력 |
| 예방접종 도우미 | 접종만 | 대시보드 통합 + 소아과 연계 |
| 네이버 날씨 | 날씨만 | 육아 맥락에서의 판단 |
| **통합 서비스** | **없음** | **이 조합 자체가 엣지** |

→ **각각의 경쟁자는 있지만, 7개를 한 화면에 = 경쟁자 제로**

```
Phase 4: dodam.life 포털 — 옴니버스 통합
```
