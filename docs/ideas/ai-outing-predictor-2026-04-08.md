# AI 외출 판단 엔진 — "오늘 나갈까?" AI 고도화 아이디어 브리프

> 아이디어 브리프 | 2026-04-08
> "단순 점수 공식이 아닌, 학습하고 예측하는 AI가 '오늘 아이 데리고 나갈까?'를 판단한다"

---

## Executive Summary

"오늘 나갈까?" 서비스의 외출 점수를 **고정 공식(Rule-Based)**에서 **예측형 AI 엔진(Hybrid ML + LLM)**으로 진화시킨다. 공공 API 데이터 + 사용자 피드백 + 지역 커뮤니티 데이터를 학습하여, "지금" 뿐 아니라 **"앞으로 어떻게 변할지"를 예측**하는 것이 핵심. 이것이 미세미세/날씨날씨가 절대 따라올 수 없는 경쟁 해자가 된다.

---

## 1. 왜 AI인가 — 경쟁 해자 분석

### Rule-Based (현재 설계) vs AI 예측

| 항목 | Rule-Based | AI 예측 엔진 |
|------|-----------|-------------|
| 점수 산출 | 고정 가중치 공식 | 학습된 가중치 + 컨텍스트 반영 |
| 미래 예측 | 불가 | "오후 3시부터 나빠져요" 가능 |
| 개인화 | 불가 | "우리 아이는 미세먼지에 민감" 반영 |
| 지역 특화 | 불가 | "우리 동네는 오후에 바람 불면 나아져요" |
| 모방 난이도 | **쉬움** (공식 공개되면 끝) | **어려움** (데이터+모델+피드백 루프) |

### 해자의 3중 구조

```
Layer 1: 데이터 해자
  → 사용자 피드백 데이터가 쌓일수록 예측 정확도 ↑
  → 경쟁자가 같은 데이터를 모으려면 같은 사용자 수 필요
  → 선점 우위 = 시간이 지날수록 격차 확대

Layer 2: 모델 해자
  → 공공 데이터 + 행동 데이터 + 지역 데이터 조합의 학습 모델
  → 단순 API 조합으로는 재현 불가
  → 도메인 특화 피처 엔지니어링이 핵심

Layer 3: 네트워크 해자
  → "우리 동네 부모 73%가 오늘 실내 활동 선택"
  → 사용자가 많을수록 지역 데이터 품질 ↑
  → 사용자 → 데이터 → 정확도 → 더 많은 사용자 (플라이휠)
```

## 2. AI 엔진 아키텍처 — 하이브리드 설계

### 전체 흐름

```
[입력 데이터]                    [AI 엔진]                    [출력]

공공 API ─────┐                ┌─────────────┐
 - 에어코리아   │               │             │              85점
 - 기상청       ├──→ Feature ──→│  경량 ML     │──→ 점수 ───→ +
 - 질병관리청   │    Pipeline   │  (Edge/CF)   │              "오늘 오전은 좋지만
                │               │             │               오후 3시부터
사용자 데이터 ──┤               └──────┬──────┘               미세먼지 나빠져요.
 - GO/STOP 피드백│                     │                       등원 때 나가세요!"
 - 후회/만족    │                     │ 복잡한 케이스
                │               ┌─────▼──────┐
지역 데이터 ────┘               │   LLM      │──→ 자연어 요약
 - 집단 판단                    │  (Cloud)    │
 - 지역 패턴                    └─────────────┘
```

### Layer 1: 경량 ML 모델 (기본 판단)

```yaml
모델_타입: Gradient Boosting (XGBoost/LightGBM) 또는 경량 신경망
추론_위치: Cloudflare Workers / Edge Runtime
응답_시간: < 100ms
비용: 거의 0 (Edge 추론)

입력_피처:
  실시간:
    - PM2.5, PM10 현재값 + 6시간 추이
    - 기온, 체감온도, 습도, 풍속
    - 자외선 지수
    - 강수 확률, 강수량
    - 지역 감염병 발생 건수 (주간)
  시간_피처:
    - 시간대 (아침/오후/저녁)
    - 요일 (평일/주말)
    - 월 (계절성)
  예측_피처:
    - 기상청 단기예보 (3시간~48시간)
    - 미세먼지 예보 (오늘/내일)
    - 감염병 주간 추이 (증가/감소/유지)

출력:
  - 외출 점수 (0~100)
  - 시간대별 점수 예측 (3시간 단위, 24시간)
  - 최적 외출 시간대 추천
  - 주요 위험 요소 (Top 3)
```

### Layer 2: LLM (자연어 요약 + 복잡한 케이스)

```yaml
모델: Claude Haiku 또는 GPT-4o-mini
추론_위치: Cloud (서버사이드)
호출_조건:
  - 항상: 점수에 대한 한줄 자연어 요약 생성
  - 선택: 복잡한 날씨 상황 해석 (ex: 황사+비+기온 급변)
비용: 요청당 ~$0.001 (Haiku 기준)

프롬프트_구조: |
  당신은 0~5세 아이를 키우는 부모를 위한 외출 판단 어시스턴트입니다.

  [데이터]
  - 위치: {지역}
  - 현재: PM2.5 {값}, 기온 {값}°C, 자외선 {값}
  - 예보: {3시간 후}, {6시간 후}
  - 감염병: {지역 유행 상황}
  - AI 점수: {점수}/100
  - 위험 요소: {top3}

  [규칙]
  - 한 문장으로 요약 (40자 이내)
  - 부모가 즉시 행동할 수 있는 정보만
  - 시간 변화가 있으면 반드시 언급
  - 예: "오전 등원은 OK, 오후 3시부터 미세먼지 주의"

출력_예시:
  - "등원 시간대 최적! 오후엔 실내 놀이 추천해요"
  - "미세먼지 '나쁨' 예상, 마스크 필수에요"
  - "수족구 급증 지역이에요. 손 씻기 꼭 챙기세요"
  - "비 올 확률 80%, 우비+장화 챙기세요"
```

## 3. 데이터 플라이휠 — 경쟁 해자의 핵심

### 3단계 데이터 수집 전략

```
Phase 1 (런칭~3개월): 공공 API Only
  → Rule-Based 점수 + LLM 요약
  → 사용자 피드백 UI 추가 ("오늘 판단 맞았나요?" 👍👎)
  → 피드백 데이터 축적 시작

Phase 2 (3~6개월): 피드백 학습 시작
  → 축적된 피드백으로 ML 모델 첫 학습
  → "이 조건에서 GO 한 부모의 80%가 만족" → 가중치 조정
  → Rule-Based → ML 모델로 전환

Phase 3 (6개월~): 풀 플라이휠
  → 지역별/시간대별 패턴 학습
  → "강남구는 오후에 미세먼지가 나아지는 패턴" 자동 발견
  → 사용자 많을수록 정확도 ↑ → 더 많은 사용자 유입
```

### 피드백 수집 UX

```
┌────────────────────────────────┐
│  오늘 외출 어땠나요?            │
│                                │
│  [😊 잘 다녀왔어요]  [😐 그냥저냥]  [😰 후회해요]  │
│                                │
│  (선택) 뭐가 문제였나요?       │
│  □ 미세먼지가 생각보다 나빴어요  │
│  □ 날씨가 갑자기 변했어요       │
│  □ 아이가 감기 걸렸어요         │
│  □ 자외선이 너무 강했어요       │
└────────────────────────────────┘
```

- **수집 시점**: 오후 6시 푸시 알림 (등원 후 귀가 시간대)
- **응답률 목표**: 20%+ (이모지 원탭으로 낮은 허들)
- **데이터 구조**: `{date, location, score, weather_snapshot, feedback, detail_tags}`

## 4. AI 예측 기능 상세

### 4-1. 시간대별 예측 (킬러 피처)

```
현재 → 3시간 후 → 6시간 후 → 12시간 후

┌──────────────────────────────────────┐
│  시간별 외출 점수                      │
│                                      │
│  🕖 7시   ████████████░░ 85점 👍      │
│  🕙 10시  ██████████░░░░ 72점 👌      │
│  🕐 1시   █████░░░░░░░░░ 38점 ⚠️     │
│  🕓 4시   ████░░░░░░░░░░ 28점 🚫      │
│                                      │
│  💡 "등원 때 나가면 최적! 오후엔      │
│      미세먼지 '매우 나쁨' 예상"       │
└──────────────────────────────────────┘
```

**예측 근거 데이터**:
- 기상청 단기예보 (3시간 단위)
- 에어코리아 미세먼지 예보 (오늘/내일)
- 감염병 주간 추이 → 당일 위험도 추정
- **과거 패턴 학습**: "이 지역에서 이 기상 조건이면 오후에 PM2.5가 X만큼 상승하는 패턴"

### 4-2. 최적 외출 시간 추천

```
🎯 오늘의 골든타임: 오전 7~10시

이유:
- 미세먼지: 오전 '좋음' → 오후 '나쁨' 전환 예측
- 자외선: 오전은 '보통', 정오 이후 '높음'
- 기온: 현재 18°C, 오후 최고 24°C (활동하기 좋아요)
```

### 4-3. 개인화 예측 (Phase 2+)

```yaml
# 개인화 피처 (피드백 축적 후)
child_sensitivity:
  dust: high    # "미세먼지 나쁨에서도 GO했는데 후회" 반복 → 민감 판정
  cold: medium  # "15°C에서 괜찮았다" 피드백 → 보통
  uv: low       # "자외선 높음에서도 만족" → 덜 민감

location_pattern:
  gangnam:
    afternoon_dust_rise: 0.82  # 82% 확률로 오후에 미세먼지 상승
    weekend_congestion: 0.65   # 주말 오후 야외 혼잡도 높음
```

## 5. 기술 구현 로드맵

### Phase 1 (Day 1~3): Rule-Based + LLM 요약

```
구현:
  - 기존 가중치 점수 공식 (에어코리아 35 + 기상청 30 + UV 15 + 감염병 20)
  - Claude Haiku API로 점수에 대한 한줄 자연어 요약 생성
  - 시간대별 예보 데이터 시각화 (기상청 단기예보 활용)

비용: LLM 요약 1건 ~₩1.3 (Haiku)
  - DAU 1만 기준 월 ₩390,000

피드백 UI:
  - 이모지 원탭 피드백 (😊😐😰)
  - 데이터 축적 시작
```

### Phase 2 (Month 2~3): 경량 ML 모델 도입

```
구현:
  - 축적된 피드백 데이터로 XGBoost 모델 학습
  - Feature: 기상+대기질+시간+요일+계절+지역 → Target: 사용자 만족도
  - Rule-Based 점수 대비 정확도 개선 시 모델 전환
  - Edge Runtime 배포 (< 100ms 응답)

비용: Edge 추론 비용 거의 0
```

### Phase 3 (Month 4~6): 풀 AI 엔진

```
구현:
  - 시계열 예측 모델 추가 (LSTM/Transformer)
    → "6시간 후 미세먼지 예측" 자체 모델
  - 지역별 패턴 클러스터링
    → "강남구 패턴" vs "서초구 패턴" 자동 분류
  - 개인화 모델
    → 사용자별 민감도 프로필 자동 생성
  - 커뮤니티 데이터 통합
    → "우리 동네 부모 73% 실내 선택" 실시간 반영

비용:
  - LLM: DAU 5만 × ₩1.3 = 월 ₩1,950,000
  - ML 학습: 주 1회 재학습, GPU 비용 월 ₩100,000~300,000
  - 총: 월 ₩200~250만 (DAU 5만 기준)
```

## 7. 비용 구조

### 월간 비용 추정 (DAU 기준)

| DAU | LLM 요약 | ML 추론 | ML 학습 | 총 월 비용 |
|-----|---------|---------|---------|-----------|
| 1,000 | ₩39,000 | ₩0 | ₩0 | ~₩40,000 |
| 10,000 | ₩390,000 | ₩0 | ₩100,000 | ~₩500,000 |
| 50,000 | ₩1,950,000 | ₩0 | ₩300,000 | ~₩2,250,000 |
| 100,000 | ₩3,900,000 | ₩0 | ₩500,000 | ~₩4,400,000 |

> LLM 비용 최적화: 캐싱 전략으로 실제 비용 50~70% 절감 가능
> - 같은 지역+같은 시간대 → 캐시 히트 (30분 TTL)
> - 비슷한 기상 조건 → 템플릿 매칭으로 LLM 호출 스킵

### 비용 vs 가치

```
DAU 1만 기준:
  월 비용: ~₩500,000
  사용자당: ₩50/월

  vs.

  미세미세 광고 수익: MAU 230만 × 광고 CPM
  → 우리가 DAU 1만 확보 시 프리미엄 모델 전환 가능
  → ₩2,900/월 구독 × 전환율 5% = 500명 × ₩2,900 = ₩1,450,000/월
  → 이미 비용 커버
```

## 8. 경쟁 해자 시뮬레이션

### 미세미세가 따라오려면?

```
미세미세 현재: 미세먼지 데이터 + 날씨 기본 정보

"오늘 나갈까?" 따라하려면:
1. 감염병 API 연동 (가능, 하지만 도메인 지식 필요)
2. 아이 옷차림 추천 엔진 (새로 개발 필요)
3. AI 예측 모델 (데이터 없음 → 처음부터 축적 필요)
4. 사용자 피드백 시스템 (전체 UX 리디자인 필요)
5. 육아 도메인 전문성 (미세먼지 앱 팀에 없음)

→ 추정 따라잡기 시간: 12~18개월
→ 그 사이에 우리는 데이터 해자가 더 깊어짐
```

### 날씨날씨가 따라오려면?

```
날씨날씨 현재: 날씨 + 미세먼지 + 옷차림(성인)

"오늘 나갈까?" 따라하려면:
1. 아이 옷차림 모드 추가 (비교적 쉬움)
2. 감염병 데이터 연동 (새 도메인)
3. AI 예측 모델 (데이터 없음)
4. 육아 맥락 UX (전체 리디자인)
```
