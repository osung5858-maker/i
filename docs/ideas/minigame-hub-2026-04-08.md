# 도담 미니게임 모음 — 아이디어 브리프

> **작성일**: 2026-04-08 (v2 — 소셜 경쟁 시스템 추가)
> **상태**: Idea → Plan 대기
> **위치**: i.dodam.life 내 별도 페이지 (`/play`)
> **컨셉**: 심심풀이땅콩 미니게임 허브 + **도전장 공유 → 친구 경쟁**
> **레퍼런스**: 원 그리기 결과 카드 (공유+경쟁 UX 참고)

---

## 1. Overview

**한 줄 정의**: 임신 준비/임신/육아 중인 부모가 심심할 때 가볍게 즐기는 미니게임 모음 페이지

**컨셉**: 카카오톡 게임탭 + 숙담 앱 미니게임 느낌. 성향체크, 원 그리기, 이름 궁합, 뭐든지 다 모아두는 잡동사니 놀이터.

**핵심 목적**:
- 부모 심심풀이 — 대기 시간에 가볍게
- 커플 소통 — 파트너와 함께 즐기기
- 스트레스 해소 — 잔잔한 힐링 게임
- 체류시간 증가 — 앱 리텐션 핵심 무기
- **바이럴 공유** — SNS 결과 카드 공유로 자연 유입 (공유 필수)

## 3. 게임별 상세 스펙

### 3.1 완벽한 원 그리기
```
[플레이]
- 빈 캔버스에 손가락으로 원 그리기
- 실시간으로 그리는 궤적 표시
- 완료 시 정확도 % 계산 (이상적 원과 비교)

[결과]
- 점수 (0~100%)
- 등급: 컴퍼스 급(95+), 달인(85+), 꽤 잘 그리네(70+), 찌그러진 감자(~70)
- 최고 기록 저장 (로컬)

[공유 카드]
- "컴퍼스는 버리셔도 되겠어요."
- "원 그리기에서 96.3점을 받았어요."
- [🔥 도전장 보내기] [다시하기] 버튼
- 도전장 받은 쪽: "OO이 96.3점! 이길 수 있을까?" → 바로 플레이 → 비교 카드
```

### 3.2 아기 이름 궁합
```
[입력]
- 엄마 이름, 아빠 이름, 아기 이름 (or 후보 이름들)

[로직]
- 획수 궁합 + 음양오행 + 자음모음 조화 (재미용, 진지하지 않게)
- 후보 이름 여러 개 비교 가능

[결과]
- 궁합 점수 (1~100)
- 한 줄 해석 ("하율이는 엄마의 따뜻함과 아빠의 용기를 물려받았어요")
- 기존 /name 페이지 연계 CTA

[공유 카드]
- "하율이와 엄마아빠 궁합 98점!"
```

### 3.3 육아 스타일 테스트
```
[문항] 5-7개 양자택일
- "아이가 넘어졌을 때?" → A: 달려간다 / B: 스스로 일어나게
- "간식 시간?" → A: 정해진 시간만 / B: 먹고 싶을 때
- etc.

[결과 유형] 4-6가지
- 자유방임형 / 헬리콥터형 / 균형잡힌형 / 학구열형 / 감성파형 / 실용파형

[공유 카드]
- 결과 유형 + 한 줄 설명 + 귀여운 일러스트
```

### 3.4 밸런스 게임
```
[질문 풀] 20-30개 (계속 추가 가능)
- "아기 성별 알고 싶다 vs 모르고 싶다"
- "모유 수유 vs 분유"
- "태교 여행 국내 vs 해외"
- "아기 이름 한글 vs 한자"
- "첫 이유식 직접 만들기 vs 사서 먹이기"
- etc.

[UX]
- 틱톡 스와이프 UX (좌/우로 선택)
- 선택 후 전체 유저 비율 표시 ("72% vs 28%")

[공유 카드]
- "나는 A파! 당신은?" + 나의 선택 요약
```

### 3.5 커플 OX 퀴즈
```
[플로우]
1. 한 사람이 "내가 좋아하는 음식은?" 등 퀴즈 출제 (자동 생성)
2. 상대방이 맞추기
3. 결과: 싱크로율 %

[질문 풀] 자동 생성 10문항
- "상대방이 제일 좋아하는 음식?"
- "상대방의 MBTI?"
- "우리가 처음 만난 곳?"
- "상대방이 가장 듣고 싶은 말?"

[공유 카드]
- "우리 커플 싱크로율 80%!"
```

### 3.6 태명 풀이
```
[입력]
- 태명 (or 아기 이름)
- 예정일 or 생년월일 (선택)

[로직]
- 태명 글자 분석 (한글 자음/모음 의미 부여)
- 사주풍 해석 (재미용)
- 태명에 담긴 소원 해석

[결과]
- "하율이는 '큰 빛을 품은 아이'라는 뜻이에요"
- 태명 의미 + 성격 예측 + 행운의 색/숫자

[공유 카드]
- 태명 + 의미 해석 + 도담 브랜딩
```

### 3.7 반응속도 테스트
```
[플레이]
- 화면 빨간색 → 기다리기 → 초록색 변경 → 터치!
- 3-5회 반복 → 평균 반응속도 ms

[결과]
- 평균 반응속도 (ms)
- 등급: 번개손(~200ms), 빠른 편(~300ms), 보통(~400ms), 느긋한 편(400+ms)
- "이 속도면 아기가 떨어뜨린 젖병도 잡을 수 있어요!"

[공유 카드]
- "나의 반응속도 187ms! 번개손이에요"
```

## 4. 기술 요구사항

| 항목 | 방식 |
|------|------|
| **라우트** | `/play` (게임 허브), `/play/[game-id]` (개별 게임) |
| **프레임워크** | 기존 Next.js + React, 별도 라이브러리 최소화 |
| **원 그리기** | Canvas API (터치 이벤트 → 좌표 수집 → 원 정확도 계산) |
| **반응속도** | `performance.now()` 또는 `Date.now()` |
| **테스트류** | 정적 JSON 데이터 + 간단한 점수 로직 |
| **공유 기능** | html2canvas → 결과 카드 이미지 생성 → 카카오톡/인스타/이미지저장 |
| **랭킹** | Supabase — 원 그리기, 반응속도 등 랭킹 저장 (선택적) |
| **도전장 시스템** | Supabase `challenges` 테이블 — `{id(UUID), game_id, challenger_name, score, metadata, created_at, expires_at}` |
| **비로그인 플레이** | 도전장 링크 → 닉네임(선택) → 즉시 플레이. 가입 강요 없음 |
| **결과 비교 카드** | html2canvas → 도전자 vs 나 점수 나란히 비교 이미지 생성 |
| **밸런스 게임 통계** | Supabase — 전체 유저 선택 비율 집계 |
| **커플 OX** | 로컬 2인 플레이 (한 기기에서 번갈아) or 링크 공유 |
| **OG 태그 동적 생성** | Next.js `generateMetadata` — 도전장 링크마다 "OO이 96.3점! 이길 수 있을까?" OG 카드 |
| **추가 비용** | ₩0 (전부 클라이언트 처리, AI 미사용. Supabase 도전장 데이터는 기존 무료 tier 내) |

## 5. 소셜 경쟁 시스템 (핵심 — "나 이만큼 했는데 넌?")

### 5.1 도전장 플로우 (Challenge Flow)

```
[A가 게임 완료]
    ↓
결과 화면: "96.3점! 도전장 보내기 🔥"
    ↓
[도전장 공유] → 카카오톡 / 링크 / 이미지 저장
    ↓
────────── 친구 B가 받는 쪽 ──────────
    ↓
[링크 클릭] → 도담 도전 페이지 진입 (비로그인 OK)
    ↓
"A가 96.3점으로 도전장을 보냈어요! 이길 수 있을까?"
    ↓
[바로 플레이] (회원가입 없이 즉시)
    ↓
[B 결과 나옴]
    ↓
┌───────────────────────────────────┐
│     🏆 결과 비교 카드              │
│                                   │
│  A (도전자)    vs    B (나)        │
│   96.3점              91.7점      │
│   컴퍼스 급            달인         │
│                                   │
│  "아깝다! 4.6점 차이로 졌어요 😭"   │
│                                   │
│  [나도 도전장 보내기]  [다시 하기]   │
│  [도담 앱 구경하기 →]              │
└───────────────────────────────────┘
```

### 5.2 도전장 기술 스펙

| 항목 | 구현 |
|------|------|
| **도전 ID** | UUID — URL 파라미터로 전달 (`/play/circle-draw?challenge=abc123`) |
| **도전 데이터** | Supabase `challenges` 테이블: `{id, game_id, challenger_name, score, created_at}` |
| **비로그인 플레이** | 닉네임만 입력 (선택) → 게임 즉시 시작. 가입 강요 없음 |
| **결과 비교** | 도전 ID로 원래 점수 조회 → 새 점수와 나란히 비교 |
| **공유 카드** | html2canvas → 비교 카드 이미지 생성 (A vs B 나란히) |
| **만료** | 도전장 7일 후 만료 (스토리지 절약) |

### 5.3 게임별 도전 적용

| 게임 | 도전 가능 | 비교 방식 |
|------|----------|----------|
| 원 그리기 | ✅ | 점수 vs 점수 (높은 쪽 승리) |
| 반응속도 | ✅ | ms vs ms (낮은 쪽 승리) |
| 이름 궁합 | ✅ | 같은 이름으로 궁합 → "엄마아빠 이름으로 누가 더 높은 점수?" |
| 육아 스타일 | ✅ | "너는 어떤 유형?" → 비교 재미 |
| 밸런스 게임 | ✅ | "A는 이걸 골랐는데 너는?" → 같은지 다른지 |
| 태명 풀이 | ⚠️ 제한적 | 같은 태명으로 결과 공유 (도전보다는 공유) |
| 커플 OX | ❌ | 커플 전용 (도전보다는 내부 플레이) |

### 5.4 결과 카드 디자인

**일반 결과 카드** (혼자 플레이 후):
```

│  [귀여운 일러스트 or 그래프]   │
│                             │
│  [🔥 도전장 보내기]  [다시하기] │

### 5.5 공유 채널 (우선순위 순)
1. **카카오톡 도전장** (Kakao SDK — "OO이 도전장을 보냈어요!" 메시지 템플릿)
2. **이미지 저장** (갤러리 저장 → 인스타 스토리 등 자유 공유)
3. **링크 복사** (OG 태그 — "원 그리기에서 96.3점! 이길 수 있을까?")
4. **인스타그램 스토리** (이미지 저장 → 스토리 업로드 유도)

### 5.6 바이럴 루프 (경쟁 강화 버전)

```
A 게임 플레이 → 결과 카드 + "도전장 보내기"
    ↓
A → 카카오톡으로 B에게 공유
    ↓
B: 링크 클릭 → (비로그인) 바로 플레이
    ↓
B 결과 → A vs B 비교 카드 → "나도 도전장 보내기"
    ↓
B → C에게 도전장 → C 플레이 → B vs C 비교 → ...
    ↓
도전 체인: A → B → C → D → ... (무한 바이럴)
```

> **핵심**: 도전장을 받은 사람이 **또 다른 사람에게 도전장을 보내는** 체인 구조.
> 원 그리기 레퍼런스처럼 "나 이만큼 했는데 넌?" 심리 자극.

## 6. UX 구조

```
/play (놀잇거리 허브)
├── 상단: "놀잇거리" 타이틀
├── 카테고리 필터: 전체 | 테스트 | 게임 | 커플
├── 게임 카드 리스트
│   ├── 카드: 썸네일 + 이름 + 한줄 설명 + 플레이 수
│   ├── 🔥 인기 배지
│   └── 🆕 신규 배지
├── 기존 기능 바로가기 섹션
│   └── 운세 | 기질 테스트 | 이름 짓기 | 성장 시뮬
└── 하단: "새 놀잇거리가 곧 추가돼요!"

/play/circle-draw (개별 게임 — 일반 모드)
├── 게임 플레이 화면
├── 결과 화면
