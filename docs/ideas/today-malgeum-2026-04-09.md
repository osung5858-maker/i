## Executive Summary

**"오늘도, 맑음"**은 영유아 부모가 아침 등원 전 3초 만에 **아이 데리고 나가도 되는지** 판단하는 마이크로 웹 서비스.

날씨 + 미세먼지 + 자외선 + 감염병 → **100점 외출 점수** + **옷차림 추천**.
AI 없음. 로그인 없음. 커뮤니티 없음. 게이미피케이션 없음.
**한국 공공 API 4개 매쉬업, 그게 전부.**

> **서비스 범위: 국내 한정** — 4개 API가 모두 한국 공공데이터(에어코리아, 기상청, 질병관리청) 기반. 해외 확장은 해당 국가 API 별도 연동이 필요하므로 Phase 1 범위 밖.

빠르게 만들어서 내놓고, 사용자 반응에서 답을 찾는다.

## 2. 핵심 원칙: 담백하게

### 만드는 것

| 기능 | 설명 |
|------|------|
| 외출 점수 (100점) | 대기질35 + 날씨30 + 자외선15 + 감염병20 |
| 옷차림 추천 | 기온 기반 레이어링 가이드 (Rule-Based) |
| 감염병 현황 | 수족구/독감/수두 등 전주 대비 추이 |
| 카카오톡 공유 카드 | 외출 점수 + 핵심 정보 OG 이미지 |

### 만들지 않는 것

| 빠진 것 | 이유 |
|---------|------|
| AI 예측 | 검증 전 최적화는 낭비. 사용자 데이터 없이 AI = 의미 없음 |
| 커뮤니티 | 사용자 없이 커뮤니티 = 유령 도시 |
| 게이미피케이션 | 외출 체크에 스트릭/배지 = 억지스러움 |
| 로그인/회원가입 | 프릭션 최소화. 위치만 선택하면 바로 사용 |
| 개인화 | Phase 1에서는 불필요. 지역만 선택 |

## 3. 타겟 사용자

### Who

- **메인**: 어린이집 부모 (아이 0~5세)
- **서브**: 유치원 부모 (아이 5~7세)
- **공통 페인**: "오늘 아이 뭐 입히지?" + "나가도 되나?"

### When: 아침 07:20~07:30

```
07:20  아이 옷 입히기 ← "오늘 뭐 입히지?" 확인
07:30  등원 출발     ← "오늘 나가도 되나?" 확인
```

→ 이 10분이 골든 타임. 한 손으로 3초에 핵심 보여야 함.

## 4. 외출 점수 공식 (100점)

### 대기질 (35점)

```
PM2.5 (영유아 강화 기준):
  0~15μg   : 35점 (좋음)
  16~25μg  : 28점 (보통)
  26~35μg  : 18점 (나쁨)
  36~50μg  : 8점 (매우나쁨)
  51+μg    : 0점 (위험)
```

### 날씨 (30점)

```
기온 적정성 (15점):
  18~25°C : 15점 | 12~17/26~30 : 12점
  5~11/31~35 : 8점 | 0~4/36+ : 4점 | -5이하 : 0점

강수 확률 (10점):
  0~20% : 10점 | 21~50% : 6점 | 51~70% : 3점 | 71%+ : 0점

바람 (5점):
  0~5m/s : 5점 | 6~10m/s : 3점 | 11m/s+ : 0점
```

### 자외선 (15점)

```
UV Index:
  0~2 : 15점 | 3~5 : 12점 | 6~7 : 8점 | 8~10 : 3점 | 11+ : 0점
```

### 감염병 (20점)

```
지역 내 주요 영유아 감염병 유행도:
  유행 없음 : 20점 | 1건 보통 : 15점
  1건 주의보 : 10점 | 2건+ 주의보 : 5점 | 경보 : 0점

대상: 수족구, 독감, 수두, 노로바이러스, RSV, 로타바이러스
```

### 판정

```
80~100점 : "오늘도, 맑음 ☀️" (초록)
50~79점  : "조심해서 나가세요 🌤" (노랑)
0~49점   : "오늘은 실내 놀이 ☁️" (빨강)
```

## 5. 옷차림 추천 (Rule-Based)

```
25°C+     : 반팔 + 반바지
20~24°C   : 반팔 + 긴바지 (or 얇은 긴팔)
15~19°C   : 긴팔 + 얇은 겉옷
10~14°C   : 긴팔 + 두꺼운 겉옷 (점퍼/가디건)
5~9°C     : 기모 내의 + 패딩
0~4°C     : 기모 내의 + 두꺼운 패딩 + 목도리
-5°C 이하  : 풀 무장 (기모내의+패딩+모자+장갑+목도리)

보정:
  강수 시 → "우비/우산 챙기세요"
  바람 강할 때 → "바람막이 추천"
  UV 높을 때 → "모자 + 선크림"
  일교차 10°C+ → "겉옷 꼭 챙기세요 (낮에 더워요)"
```

## 6. API (4개, 전부 무료)

| API | 용도 | 캐시 TTL | 호출 제한 |
|-----|------|----------|----------|
| 에어코리아 대기오염정보 | PM2.5, PM10, 오존 | 30분 | 500건/일 |
| 기상청 단기예보 | 기온, 강수, 바람, 하늘 | 1시간 | 10,000건/일 |
| 기상청 생활기상지수 | 자외선, 체감온도 | 6시간 | 10,000건/일 |
| 질병관리청 감염병 발생현황 | 주간 감염병 통계 | 24시간 | 1,000건/일 |

### 호출 전략

```
요청 → 캐시 있으면 즉시 응답 (0.1s)
     → 캐시 없으면 4개 API 병렬 호출 (1~2s) → 캐시 저장
```

```

├─────────────────────────────────┤

│  🦠 수족구 ⬆️주의  독감 ➡️유지   │
│                                 │
│  ┌─────────────────────────┐    │
│  │ 👕 오늘 옷차림            │    │

│  │ 🌂 오후 우산 챙기세요     │    │
│  └─────────────────────────┘    │
│                                 │
│  [카카오톡 공유]  [상세 보기]     │
│                                 │
└─────────────────────────────────┘
```

### 디자인 원칙

1. **3초 룰** — 스크롤 없이 핵심 전부 보임
2. **큰 점수** — 외출 점수가 화면 중앙 (시각적 앵커)
3. **컬러 시스템** — 초록(GO) / 노랑(주의) / 빨강(STAY)
4. **한 줄 요약** — 각 섹션은 아이콘 + 한 줄 텍스트
5. **옷차림이 마지막** — 가장 실용적 = 기억에 남는 위치

## 9. Lean 전략: 빠르게 만들고 검증

### Phase 1: 만들기 (Day 1~3)

- API 4개 연동 + 대시보드 UI + 점수 공식
- 카카오톡 공유 카드 + OG 이미지
- today.dodam.life 배포

### Phase 2: 관찰 (Week 2~4)

측정 항목:
- D1, D7, D30 리텐션 → "매일 오는가?"
- 시간대별 패턴 → "정말 아침에 몰리는가?"
- 카드별 체류 → "어떤 정보를 가장 보는가?"
- 공유율 → "바이럴이 자연 발생하는가?"

### Phase 3: 결정 (Week 4+)

| 시나리오 | 판단 기준 | 액션 |
|---------|----------|------|
| A: 매일 오는 사람 있음 | D7 > 30% | 기능 강화 (AI 등 도입 검토) |
| B: 시즌에만 온다 | 미세먼지/감염병 시즌만 트래픽 | 시즌 서비스로 피벗 |
| C: 감염병 반응 강함 | 감염병 카드만 클릭 | 감염병 알림 서비스로 피벗 |
| D: 아무도 안 온다 | D7 < 5% | 버리고 Day 2 서비스로 이동 |

### 성공 기준

```
D7 리텐션 > 30%
```
