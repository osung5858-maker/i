## Executive Summary

**한 줄 결론: "날씨+미세먼지+자외선+감염병+옷차림"을 한 화면에 통합한 서비스는 국내에 없다.**

- **서비스 범위: 국내 한정** — 4개 API(에어코리아/기상청/질병관리청)가 모두 한국 공공데이터 기반
- TAM 300~350만 부모, SAM 150~200만 MAU, SOM Year 1 목표 10~30만 MAU
- 국내 경쟁사 10개 앱 분석 → 7축 비교에서 전 요소 커버하는 앱 0개
- 수요 신호 17건 수집, Lead Score 8.0+ 5건 → "고수요-저충족" 영역 확인
- 해외 벤치마크 → 참고용 (ninaru baby 등). 글로벌 확장은 Phase 1 범위 밖
- **블루오션 판정: 명확 (국내 시장 기준)**

### 2-2. 개별 경쟁사 상세

#### 미세미세
- 평점 4.8 (49,000+ 평가), 누적 1,450만 DL
- 강점: WHO 기준 8단계, 직관적 UI
- 약점: 옷차림/감염병/육아 기능 전무
- 불만: 위젯 업데이트 지연, 긴 광고

#### 날씨날씨
- 평점 4.7 (39,000+ 평가), 미세미세 동일 개발사
- 강점: AI 옷차림 추천, 미세먼지+UV 통합
- 약점: 감염병/육아 없음, AI 옷차림 UX 퇴보 불만

#### 웨더핏 (한국, com.moonui.weatherfit)
- Android 전용, 인디 개발자
- 강점: **유일한 아기 옷차림 모드** (남성/여성/아기)
- 약점: 소규모 사용자, 미세먼지/UV/감염병 없음

#### 기상청 날씨알리미
- **평점 3.5** (경쟁사 대비 최저)
- 약점: UX 최악, "기상 망명족" 현상 (해외 앱으로 이탈)

### 2-3. 구조적 공백

```
시장이 완전히 분절되어 있음:

미세먼지 전문 (미세미세, IQAir) → 육아 무관
날씨+옷차림 (날씨날씨) → 성인 중심, 감염병 없음
아기 옷차림 (웨더핏) → 날씨/미세먼지 없음
공식 날씨 (기상청) → UX 최악, 생활 연계 없음
육아 기록 (마미톡, 베이비빌리) → 환경 정보 전무

→ "오늘 아이 데리고 나가도 될까?" 판단에 앱 2~3개 필요
→ "오늘도,맑음"이 한 화면으로 통합하면 카테고리 자체를 새로 정의
```

### 시장 갭 요약

1. **"판단" 기능 부재**: 기존 앱은 "정보" 제공, "나가도 되나?" 판단은 부모 몫 → 정보→행동 전환 갭이 핵심 기회
2. **아이 전용 기준 부재**: 모든 앱이 성인 기준. 영유아 별도 기준 적용 앱 없음
3. **정보 파편화**: 미세먼지+기온+UV+강수+옷차림을 한번에 볼 수 없음

### 4-2. 가장 유사한 서비스: ninaru baby (일본)

- 일본 엄마 1/3이 사용하는 육아앱
- 2018년 날씨+아이 옷차림 예보 추가 (우편번호 기반)
- 열사병 경고, UV 정보 제공
- **한계**: 미세먼지(PM2.5) 없음, 감염병 없음, 외출 Go/No-Go 판정 없음

### 4-3. 성공/실패 사례

| 서비스 | 결과 | 핵심 원인 |
|--------|------|-----------|
| **ninaru baby** | 성공 (일본 1/3 엄마 사용) | 무료 + 매일 메시지 = 습관 형성 |
| **BreezoMeter** | 성공 (Google $2.25억 인수) | B2B API 모델이 핵심 |
| **IQAir** | 성공 (지속 성장) | 하드웨어+소프트웨어 생태계 |
| **Plume Labs** | **실패** (서비스 종료) | 하드웨어 의존($179), 수익 모델 부재 |

**핵심 교훈**: 매일 열게 만드는 습관 형성 메커니즘이 핵심. 데이터 나열보다 "행동 가능한 한 줄 결론"이 사용자에게 중요.
